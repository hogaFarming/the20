Github + Jekyll搭建博客

发表于 2017-09-21

为什么从现在开始就应该写博客？
——博客能记录有价值的思考，写博客能让你面对自己，整理自己。So,BLOG UP!

Github传说中全球最大同性交友网站，那里生活着一群快乐的蓝精灵…嘻嘻开个玩笑~~
通过github和Jekyll，我们可以轻松搭建属于自己的blog，原理参照阮一峰的文章：搭建一个免费的，无限流量的Blog—-github Pages和Jekyll入门，可以用下面一张图示意：

下面简单梳理一下步骤：

1.本地初始化仓库，创建一个没有父节点的gh-pages分支，因为github规定只有在该分支下的文件，才会用Jekyll生成网页，所以blog源文件都要放在gh-pages分支下

git checkout --orphan gh-pages

2.按照Jekyll文档，编写blog网站源文件，调试方法在下文有详细说明

3.在github上建立一个空的仓库，假设名为blog_demo

4.将本地内容推送到blog_demo仓库，关于git的操作可以学习廖雪峰的Git教程

git add remote origin https://github.com/username/blog_demo.git  
git push origin gh-pages

接下来，github就会根据上传内容生成对应的网页，完成之后，通过这个地址访问blog：http://username.github.io/blog_demo/

##本地调试

###安装Jekyll程序

为了能在github上正确浏览生成的网页，最重要的步骤是在本地编写源文件并调试网站。通过Jekyll可以很轻松达到这个目的，调试好之后才能上传到github。Jekyll程序依赖于Ruby环境，因此先安装Ruby和Ruby DevKit

一、安装Ruby

选择对应版本下载

安装时路径不要出现空格,勾选Add Ruby executables to your PATH

二、安装Ruby DevKit

下载对应版本DevKit-mingw64..，进入文件夹根目录并初始化

cd c:\RubyDevKit  
ruby dk.rb init

在新生成的文件config.yml中，配置ruby路径

# config.yml #  
- C:/Ruby22-x64

安装DevKit：ruby dk.rb install

三、安装jekyll

通过Gem命令安装jekyll：gem install jekyll

###使用Jekyll调试网站

Jekyll的使用方法非常简单，进入博客目录下，运行build和serve命令即可。

这两条命令分别用来生成网站和运行本地服务器，启动成功后就能在浏览器里访问网站了，默认的端口是4000。Jekyll会实时监控目录下的文件，一旦有改动，就会立刻生效，在浏览器刷新就能看到。

Jekyll提供了模板、变量和插件功能，按照它提供的规范能轻松编写整个网站，请参照Jekyll中文文档学习使用这些功能。下面是我目前的博客目录，提供参考：

###其他

代码高亮

可以选择Pygments，为文章中的代码块提供语法高亮。
下载Python，选择2.7版本的，安装过程中选择将Python加入环境变量；
安装pip，Python包管理工具，然后安装Pygments：

python get-pip.py  
python -m pip install Pygments

在_config.yml文件中添加highlighter: pygments，为Jekyll指定高亮程序
注意还要生成css文件加载到网页中，命令为pygmentize -S monokai -f html > pygments.css
